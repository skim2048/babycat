### (2) Branch A 구조 변경 — GStreamer tee 제거 (README v0.7)

- **문제**: JetPack 6.2 호스트에서 `rtspclientsink` 플러그인 미지원 확인
- **변경 전**: GStreamer가 카메라를 수신 → tee → Branch A(MediaMTX 푸시) + Branch B(AI)
- **변경 후**: MediaMTX가 카메라를 직접 pull(Branch A). GStreamer는 MediaMTX에서 읽어 Branch B(AI)만 처리

```
카메라 ──────────────────→ MediaMTX → 클라이언트 (Branch A)
              ↓
         MediaMTX ──→ GStreamer rtspsrc → nvv4l2decoder → AI (Branch B)
```

- `config/mediamtx.yml` 신설: `source: rtsp://...` 로 카메라 pull 설정
- GStreamer 파이프라인 단순화: tee/Branch A 제거, 단일 Branch B만 유지

> OI-05 최종 확정: 디코딩 전 tee 구조 → MediaMTX pull 구조로 전환.

## 3. 파이프라인 실측 검증 결과

### 환경
- Jetson Orin NX 16GB, JetPack 6.2 (L4T R36.5.0), CUDA 12.6
- IP 카메라: H.264, 1920×1080 @ 25fps (RTSP)
- MediaMTX v1.16.3

### 검증 항목 및 결과

| 항목 | 결과 |
|---|---|
| RTSP 수신 (`rtspsrc`) | ✓ |
| GPU 하드웨어 디코딩 (`nvv4l2decoder`) | ✓ 1920×1080 H.264 |
| 포맷 변환 (`nvvidconv` → RGBA) | ✓ |
| MediaMTX 카메라 pull (Branch A) | ✓ H264 + G711 |
| Branch B 전체 (MediaMTX → GStreamer → fakesink) | ✓ |
| Python appsink (`buffer.map()` → numpy) | ✓ **25.4 FPS** |
| NVMM 버퍼 CPU 직접 접근 | ✓ (Jetson 통합 메모리 특성) |

### 검증된 Branch B 파이프라인 명령어

```bash
gst-launch-1.0 \
  rtspsrc location="rtsp://127.0.0.1:8554/live" latency=0 protocols=tcp \
  ! rtph264depay ! h264parse ! nvv4l2decoder \
  ! nvvidconv ! video/x-raw,format=RGBA \
  ! fakesink sync=false
```

### 프레임 사양 (실측)

| 항목 | 값 |
|---|---|
| 해상도 | 1920×1080 |
| 포맷 | RGBA |
| 프레임 크기 | 7.91 MB/frame (1920×1080×4 bytes) |
| 평균 FPS (appsink) | 25.4 |
| 버퍼 타입 | `nvbuf-mem-surface-array` (CPU 직접 접근 가능) |

> 참고: README의 기존 추정치 6.22 MB/frame은 RGB 기준이었으며, RGBA 실측값 7.91 MB/frame으로 정정.
