# 260323 — 범용화 방향 전환 및 구조 재편

이전 세션 파일: `260321_v1.4.md`

---

## 개요

프로젝트의 방향과 구조를 전면 재검토한 세션. 반려견 이상 행동 감지라는 특화된 목적에서 벗어나, 사용자가 정의한 조건을 감지하는 범용 AI 카메라 이벤트 감지 시스템으로 전환하기로 결정했다.

---

## 주요 결정

### 1. 범용 시스템으로 방향 전환

기존 파이프라인(RTSP → GStreamer → VLM → 이벤트 판정 → FCM)은 이미 범용 구조다. 반려견 특화 요소는 `BEHAVIORS` dict와 프롬프트 문자열뿐이며, 나머지 코드는 도메인과 무관하다. 따라서 감지 조건을 코드 외부(설정 파일 또는 UI)에서 주입할 수 있도록 범용화하기로 했다.

Wally(반려견 감지)는 이 범용 시스템 위에 조건 설정과 UI를 올린 파생 프로젝트로 구성한다.

### 2. 디렉토리 구조 재편

```
before                          after
──────────────────────────────  ──────────────────────────────
server/src/app/main.py       →  app/main.py
server/config/mediamtx.yml   →  config/mediamtx.yml
server/docker/               →  docker/
server/tests/                →  tests/
server/docker-compose.yml    →  docker-compose.yml
webapp/                      →  (제거)
```

`server/src/app/` 중첩 구조를 `app/`으로 납작하게 펼쳤다. `webapp/`은 Wally 파생 프로젝트에서 별도 구성 예정이므로 제거했다.

### 3. 버전 관리 분리

- **프로젝트(소프트웨어) 버전**: `x.y` 형식. 현재 **v1.0** (Phase 1 완료 기준).
  - v1.1 예정 시점: 감지 조건 외부화(사용자 정의 조건 주입) 완성 시
- **문서(README) 버전**: 프로젝트 버전과 별도로 관리. 현재 **v1.5**.
- **저널 파일명**: `yymmdd_vx.y.md` 형식으로 통일 (기존 `_content` suffix 제거).
