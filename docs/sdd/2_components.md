# 2. Components; 컴포넌트

본 절은 Babycat 내부 컴포넌트의 **정적 책임과 경계**를 기술한다. 인터페이스(계약·프로토콜·포트)는 §3, 동적 행위는 §4, 데이터는 §5, 설계 결정은 §6에서 다룬다. 외부 시스템/요소(***Client App***, ***RTSP Source***)는 Babycat이 소유하지 않으므로 가정하는 경계만 §2.5에 둔다.

각 컴포넌트는 **책임**(맡는 일)과 **경계**(맡지 않고 위임하거나 미결로 남긴 일)로 나누어 기술한다.

## 2.1 API Server

클라이언트의 제어·요청을 받는 외부 진입점이며, 인증과 요청 라우팅을 담당한다.

**책임**
- 인증: JWT 발급과 검증, 계정 관리(로그인·로그아웃·토큰 갱신·비밀번호 변경).
- 직접 처리: 계정, 비디오 클립·이벤트 이력의 조회·삭제, 알림 기기 등록.
- 프록시: RTSP Source 프로필 등 ***App Server*** 소관 요청을 중계.
- ***Storage*** 접근: 사용자·기기·이벤트 DB와 클립 파일을 직접 조회·관리.

**경계**
- VLM 추론·이벤트 감지 등 핵심 처리를 수행하지 않으며 ***App Server***에 위임한다.
- 라이브 미디어를 직접 서빙하지 않으며 ***MediaMTX Server***가 담당한다. 라이브 스트리밍(HLS/WebRTC)이 API를 경유하는지 여부는 §3에서 확정한다.
- 어떤 ***App Server*** 엔드포인트를 API 계약으로 노출(프록시)할지의 정확한 범위는 App–API 계약(§3, D-D)에서 정한다.

## 2.2 App Server

Babycat의 핵심 처리 컴포넌트로, 영상 분석과 그 부수 기능을 담당한다.

**책임**
- VLM 추론: ***MediaMTX Server***에서 받은 영상에 대해 추론을 수행.
- 이벤트 감지·기록: 키워드 매칭으로 이벤트를 감지하고, 해당 시점을 비디오 클립으로 ***Storage***에 저장.
- 실시간 모니터링 피드: VLM 입력 영상(MJPEG)과 분석·하드웨어 상태(SSE)를 제공.
- RTSP Source 제어: 프로필을 ***MediaMTX Server*** 소스로 적용하고, ONVIF 지원 시 PTZ를 제어.
- VLM 설정: 프롬프트·키워드·모델을 관리.

**경계**
- 외부에 직접 노출되지 않으며, ***API Server***를 통해서만 도달함을 전제한다. 현재 코드의 8080 호스트 게시는 디버그 잔재이자 제거 대상이다(F-1, D-A).
- 인증을 자체 발급하지 않고, ***API Server***가 발급한 JWT를 검증만 한다.
- 미디어의 수신·재배포는 ***MediaMTX Server***에 의존한다.

## 2.3 MediaMTX Server

라이브 미디어의 수신과 재배포를 담당하는 미디어 서버이다(기성 서드파티 컴포넌트로, Babycat은 이를 구현하지 않고 설정으로 운용한다).

**책임**
- ***RTSP Source***의 RTSP 스트림을 수신.
- 수신한 스트림을 ***App Server***(추론용)와 ***Client App***(시청용 HLS/WebRTC)에 재배포.
- ***App Server***의 제어 요청에 따라 소스 경로를 설정.

**경계**
- 영상 분석·이벤트 감지를 수행하지 않는다(***App Server*** 소관).
- 인증·계정과 무관하다. 스트리밍 접근의 인증 처리 여부는 미결이다(D-B).
- 소스·경로 설정을 스스로 결정하지 않고 ***App Server***의 제어에 따른다.

## 2.4 Storage

영속 데이터를 보관하는 수동 자원이다(`./data` 바인드 마운트 + SQLite).

**책임**
- 보관: 사용자·기기·이벤트 DB, 비디오 클립과 메타데이터, RTSP Source 프로필, 이벤트 클립 조립용 롤오버 세그먼트.

**경계**
- 스스로 통신을 개시하지 않으며, 능동 컴포넌트(***API Server***·***App Server***)의 읽기·쓰기 요청에만 응답한다.
- 접근은 네트워크가 아니라 파일시스템(파일 I/O와 SQLite)이다.
- ***API Server***와 ***App Server***가 공유한다. 두 컴포넌트의 동시 접근(App의 기록, API의 조회 등)에 관한 계약은 데이터 설계(§5, D-C)에서 정한다.

## 2.5 External Boundaries; 외부 경계

***Client App***과 ***RTSP Source***는 Babycat 외부에 있으며, 본 문서는 이들의 내부를 규정하지 않는다. 다만 다음 경계를 가정한다.

- ***Client App*** — 사용자 프론트엔드. ***API Server***(제어)와 ***MediaMTX Server***(미디어)를 통해서만 시스템과 상호작용한다.
- ***RTSP Source*** — H.264 기반 RTSP 스트림을 제공하는 외부 영상 소스. ONVIF를 지원하는 경우에 한해 ***App Server***의 PTZ 제어 대상이 된다.
