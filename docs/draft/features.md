# 세부 설계 정의

## 1. 구성

### 1.1 외부 시스템/요소

- ***Client App***
  - 사용자용 `Babycat` 프론트엔드 앱

- ***RTSP Source***
  - 라이브 비디오를 `Babycat`에 제공하는 외부 소스

### 1.2 내부 컴포넌트

- ***API Server***
  - `Babycat`의 단일 외부 진입점
  - 간단한 요청은 직접 처리하고 핵심 기능 관련 요청은 ***App Server***로 프록시

- ***App Server***
  - `Babycat`의 핵심 컴포넌트
  - VLM 추론, 키워드 매칭 기반 이벤트 감지 및 기록, 실시간 모니터링 피드를 제공

- ***MediaMTX Server***
  - `Babycat`의 미디어 서버
  - ***RTSP Source***의 라이브 비디오 자원을 적절히 분배

- ***Storage***
  - `Babycat`의 영속 데이터 저장소
  - 설정 파일이나 비디오 클립, 데이터베이스 등 제공

## 2. 기능

### 2.1 계정 관련 기능

사용자의 인증 자격을 관리하는 기능이다.

|기능|설명|
|---|---|
|1-1|로그인 할 수 있다|
|1-2|로그아웃 할 수 있다|
|1-3|로그인 상태를 유지할 수 있다|
|1-4|로그인 비밀번호를 변경할 수 있다|

### 2.2 RTSP Source 프로필 관리 기능

RTSP Source에 접속·제어하기 위한 정보(프로필)를 관리하는 기능이다.

|기능|설명|
|---|---|
|2-1|프로필을 등록할 수 있다|
|2-2|프로필을 조회할 수 있다|
|2-3|프로필을 수정할 수 있다|
|2-4|프로필을 적용/갱신할 수 있다|

등록·수정(2-1·2-3)은 프로필 저장까지만 수행하고, 저장된 프로필을 실제 스트림에 반영하는 적용(2-4)은 숨은 부작용을 피하기 위해 사용자가 명시적으로 일으키는 별도 기능으로 둔다.

### 2.3 RTSP Source 제어 기능 (Optional)

ONVIF를 지원하는 RTSP Source에 한해, App Server가 소스의 PTZ를 제어하는 기능이다.

|기능|설명|
|---|---|
|3-1|RTSP Source를 상하좌우로 움직일 수 있다(PTZ)|
|3-2|RTSP Source 이동을 멈출 수 있다|
|3-3|현재 위치를 홈으로 저장할 수 있다|
|3-4|저장한 홈 위치로 되돌아갈 수 있다|

### 2.4 라이브 스트리밍 기능

RTSP Source의 라이브 영상을 실시간으로 시청하는 기능이다.

|기능|설명|
|---|---|
|4-1|RTSP Source의 라이브 영상을 실시간으로 볼 수 있다|

라이브 영상은 HLS 또는 WebRTC 중 하나로 전달되며, 스트리밍 인증 방식은 미결이다(D-B).

### 2.5 이벤트 감지 기능

VLM으로 장면을 분석해 키워드에 해당하는 이벤트를 감지하고, 그 시점을 클립으로 저장하는 기능이다.

|기능|설명|
|---|---|
|5-1|장면 분석에 사용할 VLM 프롬프트를 설정할 수 있다|
|5-2|이벤트로 볼 키워드를 설정할 수 있다|
|5-3|사용할 VLM 모델을 선택할 수 있다|
|5-4|설정된 키워드에 해당하는 이벤트를 시스템이 자동 감지하고 클립으로 저장한다|

### 2.6 비디오 클립 관리 기능

저장된 클립과 이벤트 이력을 조회·재생·삭제하는 기능이다.

|기능|설명|
|---|---|
|6-1|저장된 클립을 조건(키워드·날짜)으로 조회할 수 있다|
|6-2|클립을 재생할 수 있다|
|6-3|클립을 선택 또는 전체 삭제할 수 있다|
|6-4|이벤트 발생 이력을 조회할 수 있다|
|6-5|이벤트 발생 이력을 삭제할 수 있다|

### 2.7 시스템 실시간 모니터링 기능

App Server가 생성하는 실시간 피드를 확인하는 기능이다.

|기능|설명|
|---|---|
|7-1|VLM에 입력되는 영상을 실시간으로 확인할 수 있다|
|7-2|분석 상태와 하드웨어 상태(온도·메모리 등)를 실시간으로 확인할 수 있다|
