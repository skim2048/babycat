# 5. 성능 요구사항 (Performance Requirements)

본 장의 성능 요구사항은 별도 명시가 없는 한 최소사양(Orin NX 16GB)을 기준 환경으로 한다. 수치가 보류된 항목은 실측 후 확정한다.

## 5.1 작업 처리량 (Throughput)

- `NFR-001` 시스템은 ***Video Source*** 1개의 H.264 비디오 스트림을 끊김 없이 실시간으로 수신하고 디코딩해야 한다. 지원하는 최대 해상도, 프레임레이트는 작성을 보류한다.
- `NFR-002` 시스템은 VLM 추론을 연속적으로 수행해야 한다. 최소 추론 처리량(시간당 추론 횟수)은 작성을 보류한다. 측정은 별도 벤치마크로 수행한다.

## 5.2 동시 세션 (Concurrent Session)

- `NFR-003` 시스템은 최소 N개의 동시 라이브 스트리밍 세션(HLS/WebRTC 합산)을 지원해야 한다. N은 작성을 보류한다.
- API 동시 요청에 대한 요구사항은 두지 않는다. 시스템은 소수 운영자의 사용을 전제한다.

## 5.3 대응 시간 (Response Time)

대응 시간이 특히 중요한 항목만 정의한다.

- `NFR-004` 이벤트 감지 지연: 이벤트 상황 발생부터 이벤트 인지(키워드 매칭)까지의 지연은 VLM 추론 주기에 종속된다. 허용 상한은 작성을 보류한다.
- `NFR-005` 라이브 스트리밍 지연: WebRTC 및 HLS의 허용 지연은 작성을 보류한다.
- `NFR-006` 시스템 기동 시간: 모델 캐시가 존재하는 상태(웜 부트)에서 기동 완료까지의 허용 시간은 작성을 보류한다. 최초 부팅의 모델 컴파일 시간은 기동 시간에 포함하지 않는다.
- `NFR-007` PTZ 명령 반영 지연: 사용자 입력부터 카메라 동작까지의 허용 지연은 작성을 보류한다.

## 5.4 성능 종속관계 (Performance Dependency)

- 이벤트 감지 지연(`NFR-004`)은 VLM 추론 주기에 비례한다. 추론 주기는 하드웨어 등급에 종속된다 — Orin NX에서 추론 1회당 약 4~5초, AGX Orin에서 약 1~2초가 관찰되었다.
- 추론 주기가 짧아지면(고성능 하드웨어) 연속 추론 간 입력 프레임의 변화량이 줄어들어, 프레임 윈도 설정의 조정이 필요할 수 있다.
- 생성 토큰 상한을 늘리면 장면 설명이 길어져 키워드 매칭 기회가 늘어나는 대신 추론 시간이 길어진다.
- 동시 스트리밍 세션 수와 VLM 추론 성능의 상호 영향은 작성을 보류한다(실측 필요).

## 5.5 그 외 성능 요구사항 (Other Performance Requirements)

- `NFR-008` 이벤트 감지 품질: 감지 정확도(precision)와 재현율(recall)의 목표치 및 측정 방법(검증 데이터셋 구성)은 작성을 보류한다. 측정에는 대체 소스를 이용한 입력 재현을 활용한다.
