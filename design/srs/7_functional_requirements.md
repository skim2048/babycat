# 7. Functional Requirements; 기능 요구사항

본 장은 기능 계층(functional hierarchy) 방식으로 조직화하며, 상위 계층은 2.3의 시나리오 구분을 따른다. 식별자·우선순위 표기는 1.3을 따른다(무표기 = `P1`).

## 7.1 사용자 인증

- `FR-001` 시스템은 사용자 이름과 비밀번호로 사용자를 인증하고, 성공 시 만료 시간이 있는 JWT 액세스 토큰과 리프레시 토큰을 발급해야 한다.
- `FR-002` 시스템은 유효한 리프레시 토큰으로 액세스 토큰을 갱신해야 한다. 사용된 리프레시 토큰은 폐기하고 새 리프레시 토큰을 발급해야 한다(토큰 회전).
- `FR-003` 시스템은 로그아웃 시 해당 리프레시 토큰을 폐기해야 한다.
- `FR-004` 시스템은 비밀번호 변경 기능을 제공해야 하며, 변경 시 해당 사용자의 모든 리프레시 토큰을 폐기해야 한다.
- `FR-005` 시스템은 초기 계정의 비밀번호가 변경되지 않은 동안 로그인 응답에 비밀번호 변경 필요를 표시해야 한다.
- `FR-006` 시스템은 무차별 대입 공격을 막기 위한 로그인 시도 제한을 적용해야 한다. 세부 정책은 6.2를 따른다.
- `FR-007` 시스템은 인증(`FR-001`~`FR-002`)과 상태 확인을 제외한 모든 API 요청에 유효한 JWT를 요구해야 한다.

## 7.2 카메라 프로필 관리

- `FR-008` 시스템은 카메라 프로필(IP, 계정, RTSP 포트, 스트림 경로, ONVIF 포트)을 저장해야 한다.
- `FR-009` 시스템은 프로필 적용 시 필수 항목을 검증하고, 검증 실패 시 적용하지 않고 오류를 반환해야 한다.
- `FR-010` 시스템은 프로필 적용 시 비디오 파이프라인을 새 소스로 재시작해야 한다.
- `FR-011` 시스템은 프로필 조회 시 비밀번호 원문을 반환하지 않아야 한다(설정 여부만 표시).
- `FR-012` 시스템은 재기동 시 저장된 카메라 프로필을 자동으로 적용해야 한다. 미디어 서버가 준비되지 않은 경우 재시도해야 한다.

## 7.3 카메라 제어

본 기능군은 RTSP Source가 ONVIF PTZ를 지원하는 경우에 한한다(2.3-B).

- `FR-013` 시스템은 ONVIF를 통해 카메라의 연속 이동(pan/tilt/zoom) 및 정지 명령을 수행해야 한다.
- `FR-014` (`P2`) 시스템은 카메라 위치 프리셋의 저장 및 이동 기능을 제공해야 한다.

## 7.4 라이브 스트리밍

- `FR-015` 시스템은 인증된 요청에 대해 스트림 접속 정보(스트림 URL과 JWT)를 발급해야 한다.
- `FR-016` 미디어 서버는 JWT 검증에 성공한 연결에만 스트림을 송신해야 한다.
- `FR-017` 시스템은 HLS와 WebRTC 두 가지 방식의 라이브 스트리밍을 제공해야 한다.

## 7.5 비디오 분석 및 이벤트 감지

- `FR-018` 시스템은 수신 중인 비디오 스트림에서 주기적으로 프레임을 추출하여 VLM에 입력하고 장면 설명 텍스트를 생성해야 한다.
- `FR-019` 시스템은 VLM 프롬프트와 이벤트 키워드(쉼표 구분 복수 키워드)의 설정 기능을 제공해야 한다.
- `FR-020` 시스템은 VLM 응답 텍스트에 이벤트 키워드가 포함되는지(대소문자 무시) 검사하고, 포함 시 이벤트를 발생시켜야 한다.
- `FR-021` (`P2`) 시스템은 후보 목록(`VLM_MODELS`) 내에서 VLM 모델 전환 기능을 제공해야 한다. 동시 로드는 지원하지 않는다(6.7 참조).

## 7.6 이벤트 클립 저장

- `FR-022` 시스템은 이벤트 발생 시 해당 시점의 비디오 클립(mp4)과 메타데이터(json: 발생 시각, 매칭 키워드, VLM 응답 텍스트)를 쌍으로 저장해야 한다.
- `FR-023` 시스템은 이벤트 발생 이력(트리거, 클립 이름, 발생 시각)을 데이터베이스에 기록해야 한다.
- `FR-024` 시스템은 저장 공간이 기준 이하로 떨어지면 오래된 클립부터 자동 삭제하여 공간을 확보해야 한다. 기준값은 작성 보류.

## 7.7 클립 조회·재생·삭제

- `FR-025` 시스템은 클립 목록 조회 기능을 제공해야 한다. VLM 응답 텍스트에 대한 부분 일치 검색, 날짜 범위 필터, 페이지네이션을 지원해야 한다.
- `FR-026` 시스템은 클립 파일을 HTTP로 스트리밍해야 하며, 구간 탐색을 위한 Range 요청을 지원해야 한다.
- `FR-027` 시스템은 클립의 선택 삭제와 전체 삭제 기능을 제공해야 한다. 클립 삭제 시 메타데이터도 함께 삭제해야 한다.

## 7.8 시스템 상태 모니터링

- `FR-028` 시스템은 VLM 추론 결과와 하드웨어 상태를 실시간으로 구독할 수 있는 인터페이스(SSE)를 제공해야 한다.
- `FR-029` (`P2`) 시스템은 VLM에 입력되는 프레임을 확인할 수 있는 영상 스트림(MJPEG)을 제공해야 한다.
